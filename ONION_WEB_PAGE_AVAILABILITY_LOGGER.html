ONION_WEB_PAGE_AVAILABILITY_LOGGER

The single web page application featured in this tutorial web page is a dark-web-hosted software application which checks whether another dark-web-hosted web page (namely, the root-level directory of karbytes’ largest .onion website) is online once every sixty seconds and keeps a log displaying information about the one hundred most recent polls (while total success and failure counts keep incrementing indefinitely and the oldest record in the table is removed to make way for each new record at the bottom of the table after the table becomes one hundred rows tall). In order to keep the polling application running instead of pausing while the respective Tor browser tab is not immediately open on mobile devices, the polling application enables the user to press a button which starts playing three sound files simultaneously (where each sound file is playing at an audible volume and where each sound file has a different play duration (such that the session stays active for an indefinitely long time due to the fact that the sound files do not stop and restart at the same times)).

A live version of the ONION_WEB_PAGE_AVAILABILITY_LOGGER application which is featured in this software engineering tutorial is available at the following .onion web page (and, like all other .onion web pages, is only accessible via the Tor browser (or other functionally similar dark web browsing technologies (if they exist))): http://qkbrwfubnh4knc6kkhx6uepccavpwezdf2al7w2quepe3qociegsi3yd.onion/KARBYTES_BLOG_APPS/ONION_WEB_PAGE_AVAILABILITY_LOGGER/onion_web_page_availability_logger.html

ONION_WEB_PAGE_AVAILABILITY_LOGGER Interface (after clicking keep_session_alive())

The screenshot image below depicts a use case of the ONION_WEB_PAGE_AVAILABILITY_LOGGER application where there are some successes and some failures logged. The failures resulted from karbytes deliberately closing the laptop computer which was serving the .onion web page being polled by ONION_WEB_PAGE_AVAILABILITY_LOGGER. Once karbytes opened the laptop again and restarted the Tor and Apache services, ONION_WEB_PAGE_AVAILABILITY_LOGGER resumed logging successful attempts to visit the target .onion web page.

(Even though both ONION_WEB_PAGE_AVAILABILITY_LOGGER and the target .onion web page are being hosted from the same top-level domain, ONION_WEB_PAGE_AVAILABILITY_LOGGER is a client-side application instead of a server-side application (meaning that ONION_WEB_PAGE_AVAILABILITY_LOGGER needs to only load one time and can keep running even while the dark web server hosting it is offline (provided that the respective ONION_WEB_PAGE_AVAILABILITY_LOGGER web browser tab stays open and, if using a mobile device to access ONION_WEB_PAGE_AVAILABILITY_LOGGER, the user clicked the keep_session_active() button at least one time))).
